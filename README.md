# Cloud Letter (云笺) - 现代开源版

一个功能丰富、极轻量且高度模块化的早安推送系统。支持通过企业微信、微信测试号或个人邮件等渠道，向您或在乎的人推送一份聚合了每日天气、优美壁纸、名言警句、纪念日倒计时的专属早报。

## 🌟 核心特性
- **全面解耦与隐私保护**：采用 `.env` 环境变量机制挂载所有的凭证方案与个人隐私数据，您的代码分享更为纯粹安全。
- **丰富的内置挂载**：默认继承封装了和风天气预报与生活质量、天行数据、金山词霸每日一句以及必应精美壁纸。
- **完善的日期计算**：不论是公历或者农历的“生日”、“纪念日”以及“我们在一起的 第xxx天”，精准计算毫无偏移。

## 🚀 快速启动

### 1. 安装项目环境依赖
环境要求 Python 3，仅需安装极其精简的核心依赖包：
```bash
pip install -r requirements.txt
```

### 2. 配置私有环境变量（核心步骤）
> [!IMPORTANT]  
> 您的所有配置项被严格隔离保护，无论您如何克隆或重置 Git，请绝不要将 `.env` 提交到公开网络！

请将项目根目录下的参考模板 `.env.example` 复制或重命名一份为 `.env`：

```bash
# Windows
copy .env.example .env
# Linux / macOS
cp .env.example .env
```
然后用任何文本编辑器打开 `.env` 文件，按需仔细填入您对应的 `CORPID`、`APPSETTING` 等参数项。

### 3. 一键运行与定时挂靠
直接在根目录执行重新组织后的主入口模块即可：
```bash
python main.py
```
> [!TIP]
> 推荐您部署于 Linux crontab 或各大云服务商的定时触发器上，设置每日早晨如 7:00 自动触发此脚本，体验绝佳自动化早安唤醒。

## ☁️ 免费云端部署 (全自动化白嫖指南)

如果您不想自己花钱购买及维护服务器，系统的代码已被深度改造，**天然支持 Serverless 结构**。为您提供三种市面上**完全免费**的云端一键挂载方案，您可以任选其一：

### 方案 A：GitHub Actions (⭐ 强烈推荐：无门槛，永久免费)
依托于 GitHub 官方机房的最极简方案：
1. **托管代码**：点击项目右上角 Fork，或将本项目代码打包上传进您的 GitHub 个人私有仓库中（注意：绝不要由于任何失误携带真实私钥的 `.env` 哦！）。
2. **挂载密钥**：进入您的仓库，从顶部菜单进入 `Settings` -> 展开侧栏 `Secrets and variables` -> 选择 `Actions`。点击 `New repository secret`，对照着您的 `.env` 将参数一个接一个加进去（如变量名写 `CORPID` 并粘贴真实企业ID）。
3. **坐享其成**：本项目已经内置了自动流水线（配置文件为 `.github/workflows/run.yml`）。一旦填好密钥，它会在**每天北京时间早上 7:00** 自动醒来拉起沙盒为您发信！您也可以切到仓库的 `Actions` 界面手工按键试运营。

### 方案 B：腾讯云函数 (SCF) / 阿里云函数计算 (FC)
如果访问海外 GitHub 比较慢，在国内云厂商里白嫖海量的 Serverless 每月免费额度是网络最稳的途径：
1. **新建函数**：登录任意主流云的控制台，搜索「云函数」，创建一个空白模版，运行环境务必选择 **Python 3.10+**。
2. **源码打包**：把咱们现在的整个文件夹压缩成 zip 包上传到云函数控制台，或者干脆直接点击它的「本地上传文件夹」按钮。
3. **注入环境**：云函数的控制界面一定有一栏「高级配置」或「环境变量」。像填表一样在这里新建配置项，Key 填 `CORPID`，Value 填密钥...重复上述动作。
4. **绑定入口与触发器**：关键一步：将云页面的 `执行方法 (执行入口)` 修改为 `main.main_handler` (针对腾讯云) 或 `main.handler` (针对阿里云)。随后在外面绑定一个「定时触发器」（如设定 `0 0 7 * * * *` 表示每一天早晨 7 点），完毕。

### 方案 C：基于 PythonAnywhere 平台
这是一个在欧美长续发展的资深全免费 Python 托管虚拟机：
1. 到官网免费注册进入 Dashboard 主屏，点击拉起一个 `Bash` 小黑窗控制台。
2. 敲击 Linux 命令 `git clone` 将本项目的克隆下发过去拉取到沙盒内。
3. 回到主屏点进顶部的 `Files` (文件管理器) 标签，找到拉取成功的目录，手写建一个真实的 `.env` 并直接填入您的隐私变量。（由于这是由于您本人独享的封闭沙箱容器，直接存放 `.env` 理论上极为安全）。
4. 退回主主菜单激活 `Tasks` 任务计划标签页。
5. 这个免费账户允许你每天挂一个脚本任务：建立一个发车项，运行命令框体敲入：`/usr/bin/python3 /home/您的英文账号名/Cloud_Letter/main.py`。
6. 最后定好几点触发并拍下 `Save` 按钮。


## 📂 项目结构概览

为了方便开源社区贡献或您本人的二次扩展开发，项目的每一层架构都已被重构剥离：

```text
Cloud_Letter/
├── main.py                    # 服务运行主入口
├── .env                       # (由您创建) 本地真实环境变量，含隐私参数 
├── .env.example               # 配置参考模板 (全公开)
├── requirements.txt           # 库依赖清单
└── cloud_letter/              # 系统核心业务包
    ├── config.py              # 配置总线总控 (动态加载 .env 映射)
    ├── core/                  # 【处理层】负责日期历法运算 (date_utils) 与卡片排版合成 (builder)
    ├── providers/             # 【接口层】调用各大外部三方平台能力 (天气、壁纸、词库等)
    ├── notifiers/             # 【推送层】实际执行消息分发 (企业微信消息、测试微信号推送、邮件)
    └── templates/             # 【视图层】包含邮件推送的 HTML 底层渲染版式
```

## 🛠️ 如何参与功能拓展
1. **添加新的数据提供商 (Providers)**：在 `cloud_letter/providers/` 目录下新增脚本，例如调用某个新星座运势 API。
2. **数据汇入版块 (Builder)**：修改 `cloud_letter/core/builder.py` 内部的合成流 `handle_msg()`，将新拉取到的数据拼接到每日早报流 `info_list` 中。
